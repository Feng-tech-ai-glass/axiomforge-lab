# Demo Plan: AI Puzzle Inventor

## 一句话
AI 现场发明一个新谜题，自己测试，自己改好，观众一眼看懂。

## 画面布局
```
┌─────────────────┬──────────────────┐
│                 │                  │
│   AI 设计区      │   测试区          │
│                 │                  │
│  生成谜题/关卡    │  Bot 尝试解题     │
│  显示设计思路     │  成功/失败动画    │
│                 │                  │
├─────────────────┴──────────────────┤
│  AI 思考过程                        │
│  "这个太简单了，加一个障碍"          │
│  "通关率 100% → 降到 60%"          │
│  "第3版：难度刚好"                  │
├────────────────────────────────────┤
│  [难度条] [版本号] [通关率] [新颖度]  │
└────────────────────────────────────┘
```

## AI 每一步做什么
1. 生成一个谜题（用世界模型预测难度）
2. 让 solver bot 尝试解（工具执行层）
3. 分析结果（抽象引擎发现规律）
4. 如果太简单/太难 → 自动修改（自改进）
5. 如果改好了 → 锁住（Answer Anchor）
6. 如果改坏了 → 回滚（记忆系统）
7. 最终输出：一个"刚好有趣"的新谜题

## 用到的 DNA 层
- 世界模型: 预测谜题难度
- MC: 决定"该加难度还是降难度"
- 记忆: 记住什么设计失败过
- 抽象: 发现"什么样的谜题更好玩"
- Answer Anchor: 锁住好设计
- Effort Scaler: 难题多花时间设计
- 自改进: 每轮设计越来越好

## 谜题类型选择（从简单到复杂）
1. 数字谜题（最快实现）: "用+-×÷组合4个数字得到24"
2. 迷宫（中等）: "生成一个可解但有挑战的迷宫"
3. Sokoban推箱子（最有视觉效果）: "设计一个新关卡"

## 建议先做: 数字谜题
- 不需要图形界面
- 终端就能演示
- 1周内能做完
- 规则简单，普通人秒懂

## 14天计划
Day 1-3:  谜题生成器 + 难度评估器
Day 4-6:  solver bot + 自动测试
Day 7-9:  修正循环（太难→简化，太简→加难）
Day 10-12: 接入 DNA 全部 7 层
Day 13-14: 包装 demo + 录屏
